2. 시스템 관점의 아키텍처 및 데이터 모델링 설계
문제를 단순한 UI/UX나 기능 단위가 아닌 시스템 전체 구조 관점에서 바라봅니다. 현직에서는 환자 신청 키오스크, 영양실 식이관리 대시보드, 차량운행 관제 대시보드, 기사 앱 등 서로 연결된 4개 서비스를 하나의 도메인으로 통합 설계하고, 한 서비스의 입력이 다른 서비스 화면에 실시간 반영되는 end-to-end 데이터 흐름을 직접 구축했습니다. 이 과정에서 Firebase·AWS를 연동한 백엔드 아키텍처를 설계하고, 구간별 이동시간 DB를 모델링해 배차 예상 소요시간을 자동 산출하는 등 데이터 모델링과 API 설계를 주도했습니다. 고객용(B2C)과 관리자/전문가용(B2B) 서비스의 비즈니스 로직 차이를 인지해 멀티 레포·클린 아키텍처로 독립적인 배포 사이클과 유지보수성을 극대화한 경험 또한 보유하고 있습니다.
